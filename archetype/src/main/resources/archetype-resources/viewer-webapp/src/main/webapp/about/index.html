This app has been generated by the tutorial quickstart archetype for Apache Isis™, a framework to let you rapidly develop domain-driven apps in Java.

The app consists of a bare minimum of classes, so that you can easily strip it back and reuse for your own purposes. It is configured with:

Component Access URL Description Further info on the Isis website
Wicket Viewer wicket/ Provides a generic UI for end-users, built with Apache Wicket™ wicket viewer
In-memory Object Store n/a Store objects in memory (non persistent) In-memory objectstore
Shiro Security n/a Provides authentication and authorization, using Apache Shiro™ shiro security

App contents

The key files in the application are:

  • Domain classes (todo functionality) (in dom module and objstore-jdo module)
    • ToDoItem domain entity
    • ToDoItems domain service (repository/factory)
  • Fixture classes (in fixture module)
    • ToDoItemsFixture example fixture data
    • ToDoItemsFixtureService domain service to install fixtures at run-time
  • WEB-INF/web.xml (in viewer-webapp module)
    • AboutPageFilter - filter that redirects to this page
    • ResourceCachingFilter and ResourceServlet - filter and servlet for serving up static resources.
    • WicketFilter - filter for the wicket application, mapped to /wicket/*
  • WEB-INF/isis.properties (in viewer-webapp module)
    • isis.services - specifies the two domain services (ToDoItems, ToDoItemsFixtureService)
    • isis.persistor - specifies to use the in-memory object store
    • isis.authentication and isis.authorization - security mechanism to use (bypass effectively disables)
  • WEB-INF/shiro.ini (in viewer-webapp module)
    • configuration for Shiro security

Next steps

Use the following as a check-list for refactoring towards your own application:

  • Experiment with users, roles and permissions (in shiro.ini)
  • Refactor the ToDoItem and ToDoItems towards your own application's functionality; obviously you are likely to introduce many more classes and services
  • Add persistency to your application. Tutorial 2 explains all about this.
  • Update this page (about/index.html) as required for your application, and/or remove the welcome-file-list from web.xml
